If the same alias appears more than once, the first enabled row with a valid canonical plan wins.

Apply alias normalization to both booking and credit `plan` values before matching. The canonical plan names still map to themselves. Unknown aliases are not eligible for matching. Matched report rows must emit the canonical plan value, never the raw alias. Unmatched rows still leave `plan` blank. Signal completion by leaving the regenerated files in `/app/out`.

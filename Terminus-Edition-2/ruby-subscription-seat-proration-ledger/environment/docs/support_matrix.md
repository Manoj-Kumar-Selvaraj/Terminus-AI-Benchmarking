# Support Matrix

Canonical kinds: BASIC, PRO, ENT. Legacy alias rows live in `/app/config/kind_aliases.csv`; the three BASIC/PRO mappings are usually enough for day-to-day batches.

Later milestones add policy and calendar controls. `/app/config/kind_policy.csv` controls which canonical kinds are enabled and their numeric priority for `ANY` corrections. `/app/config/release_calendar.txt` controls which source and action dates are open for release processing.

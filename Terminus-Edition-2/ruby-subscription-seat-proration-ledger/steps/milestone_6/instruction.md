Finish the subscription seat proration reconciler in `/app/app/reconcile.rb` by adding contract-rate proration controls from `/app/config/proration_contracts.csv`. Keep every rule from milestones 1 through 5, including release-calendar dates, seat-ledger capacity, enabled-kind policy, `ANY` ranking, realtime windows, row-position consumption, correction input order, and the existing `/app/out/seat_credit_report.csv` and `/app/out/seat_credit_summary.txt` schemas.

The contract file has columns `subscription_id,period_start,period_end,kind,cutover_date,rate_cents,rounding_mode,enabled`. Trim all fields. A valid row has a nonblank subscription id, canonical concrete kind (`BASIC`, `PRO`, or `ENT` after alias normalization), real `YYYYMMDD` period start/end/cutover dates, `period_start <= cutover_date <= period_end`, nonnegative integer `rate_cents`, enabled `Y`, `YES`, or `1`, and rounding mode `FLOOR`, `CEIL`, or `NEAREST` case-insensitively. Disabled, malformed, alias-named-only without canonicalization, unsupported-kind, impossible-date, out-of-period cutover, negative-rate, or unknown-rounding rows create no contract eligibility.

For each candidate source row, derive the source date from its `source_ts`. The source date must fall inside at least one valid contract row for the same subscription and canonical source kind. If several rows match, choose the row with the latest `cutover_date`; if that still ties, choose the later input row from `proration_contracts.csv`. Compute the expected credit amount from that chosen contract as `rate_cents * remaining_days / period_days`, where `remaining_days` counts calendar days from the source date through `period_end` inclusive and `period_days` counts calendar days from `period_start` through `period_end` inclusive. Apply the chosen row's rounding mode: `FLOOR` rounds down, `CEIL` rounds up, and `NEAREST` rounds half up. The source/action amount must equal this expected amount as an integer before the source row is eligible.

Apply this proration gate after milestone 5 calendar and ledger eligibility, but before final candidate ranking. For `ANY`, capacity-exhausted rows and rows whose amount does not equal the selected contract proration must be excluded before ranking by latest `source_ts`, policy priority, and source row. A rejected correction must not consume source rows or seat-ledger capacity.

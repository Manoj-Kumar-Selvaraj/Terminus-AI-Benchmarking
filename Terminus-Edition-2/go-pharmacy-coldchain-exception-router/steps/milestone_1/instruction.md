The pharmacy coldchain team needs a Go reconciler at `/app/cmd/reconcile/main.go` that pairs exception rows from `/app/data/exceptions.csv` with source accessions in `/app/data/accessions.csv`. Window timing comes from `/app/config/windows.csv`.

Start with the baseline matching rules before any alias or window logic is added. Treat a correction as matched only when shipment id, pharmacy id, chain id, depot, and amount all agree on the same source row, the source status is `RECEIVED`, the correction reason is `SPLIT`, `TEMPBREACH`, or `RECHECK`, and package type is a canonical `CHEM` or `HEME` value. Both timestamps must be numeric, the correction time must be on or after the source scan time, and each source row can be consumed once.

Write results to `/app/out/coldchain_exception_report.csv` with columns `exception_id`, `shipment_id`, `pharmacy_id`, `chain_id`, `package_type`, `amount`, `reason`, and `status`. Matched rows should show the canonical source package type. Unmatched rows leave package type blank. Also write `/app/out/coldchain_exception_summary.txt` with positive integer totals for matched and unmatched counts and amounts. Preserve correction input order and use only `MATCHED` or `UNMATCHED` in the status column.

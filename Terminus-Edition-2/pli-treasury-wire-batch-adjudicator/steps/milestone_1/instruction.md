The treasury wire PL/I adjudicator is used for same-day wire exception clearing after a control-deck refresh. The public harness entrypoint is `/app/scripts/run_batch.sh`. Repair `/app/src/wire_batch.pli`, `/app/src/wire_rules.pli`, or the batch harness so `/app/data/claims.psv` adjudicates against `/app/data/clearing.psv` and regenerates `/app/out/wire_report.csv` plus `/app/out/wire_summary.txt` on every run. When needed, `/app/scripts/pli_wire.awk` may also require adjustment.

Claims require full agreement on `wire_id`, `account`, `amount_cents`, `rail_code`, and `branch_id`; prefix or substring matches are not enough. For `rail_code`, trim and case-fold alias keys and both clearing and claim values **for comparison only**. Each `ALIAS_*` declaration in `/app/src/wire_rules.pli` has the form `raw=>canonical`; the right-hand canonical token keeps its declared spelling in cleared report output (for example `F=>FED` emits `FED`, not `fed`). Unknown values do not match a different canonical value.

A clearing row is eligible only when its `state` equals runtime `ELIGIBLE_STATE`. A claim is eligible only when its trimmed `reason_code` equals runtime `REASON_1`, `REASON_2`, or `REASON_3`, case-insensitively. Load `ELIGIBLE_STATE`, `REASON_*`, `NEGATIVE_REASON_CODES`, and alias declarations from `/app/src/wire_rules.pli` at runtime; do not hardcode sample values from the shipped environment.

Each clearing row may be consumed once, and claim input order must be preserved in the report. When multiple unused rows qualify, choose the latest `posted_ts`; if timestamps tie, choose the earliest clearing input row. Claims whose reason appears in comma-delimited `NEGATIVE_REASON_CODES` must carry a negative `amount_cents`; all other eligible reasons must carry a positive amount. Totals always use absolute claim amounts.

Write `/app/out/wire_report.csv` with the exact pipe-delimited columns `claim_id|wire_id|account|branch_id|rail_code|amount_cents|reason_code|status`. Status is exactly `CLEARED` or `RETURNED`; returned rows leave `rail_code` empty, and cleared rows emit the canonical rail. Write `/app/out/wire_summary.txt` with integer keys `cleared_count`, `cleared_amount_cents`, `returned_count`, and `returned_amount_cents`.

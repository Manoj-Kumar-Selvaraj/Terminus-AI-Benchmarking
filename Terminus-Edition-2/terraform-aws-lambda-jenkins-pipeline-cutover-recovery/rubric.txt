# Rubric 1
Agent restores the complete twelve-stage Lambda deployment in the documented Jenkins-equivalent order, +5
Agent preserves the pinned Terraform AWS Lambda module source and version, published package identities, versioned live aliases, and distinct function identity for every stage, +5
Agent enforces the exact documented timeout, memory, reserved concurrency, runtime, and least-privilege permission contract for each stage, +5
Agent preserves execution, batch, item, and artifact identity through a normal end-to-end batch run, +3
Agent rejects missing, duplicated, or reordered stage names, wrong module/runtime choices, wildcard actions, wildcard principals, and shared or unversioned stage resources before recording a deployment, +5
Agent edits the trusted runtime, writes runtime state directly, invokes `$LATEST`, or replaces the fleet with one shared Lambda to force success, -5
Agent accepts duplicated stage names, wildcard permissions, changed stage resources, or an incomplete stage graph, -5
Agent loses request identity or hardcodes successful runtime output instead of repairing the deployment loader and Terraform contract, -5
# Rubric 2
Agent preserves milestone 1 deployment, identity, resource, and permission behavior while adding durable retry recovery, +5

Agent replays completed external work, starts recovery from `intake`, or treats completed execution reruns as new work, -5
Agent disables failure injection, suppresses errors, or fabricates checkpoints rather than preserving durable progress, -5
Agent regresses the Lambda stage graph, permissions, runtime contract, or identity preservation from milestone 1, -5
# Rubric 3
Agent preserves prior deployment and retry behavior while isolating permanently invalid items from valid siblings, +5

Agent enforces one execution owner per batch while allowing unrelated batches to acquire independent per-batch locks, +5
Agent releases locks after successful and partial completion and keeps repeated partial submissions idempotent, +3
Agent deletes invalid records, marks every item successful, or blocks valid siblings behind poison work, -5
Agent uses one global lock, disables concurrency, or allows a second execution owner to publish the same batch, -5
Agent regresses durable retry checkpoints, idempotency, deployment identity, or prior output contracts, -5

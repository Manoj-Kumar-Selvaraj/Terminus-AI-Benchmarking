Finish the wire-return batch in `/app/src/wire_returns.cbl` for settlement-cycle controls. Wire records may include an 8-character settlement date in positions 36–43 immediately after the settled-status field, and return records may include an 8-character return date in positions 32–39 immediately after the account field. Use `/app/config/cycle_calendar.txt`, where each line is `YYYYMMDD STATUS` and only rows marked `OPEN` (case-insensitive) are settlement-cycle days.

A return can clear only when the wire settlement date and return date are nonblank, both appear in the calendar with status OPEN (not merely listed), the return date is not before the wire settlement date, and the return arrives within two open cycle days after the wire settlement date. Count open days after the wire settlement date through the return date, including the return date if it is open and excluding closed or absent calendar dates. Exactly two open cycle days is still eligible; three or more is not. Dates missing from the calendar are not open, and a blank settlement date or blank return date must be treated as ineligible rather than as a same-day match. If more than one unused wire satisfies all matching and cycle-window rules for a return, clear against the eligible wire with the latest settlement date; when dates tie, use the earliest wire input row (the matching wire that appears first in `wires.dat`, not the return input row). Keep all prior behavior and output schemas for `/app/out/wire_return_report.csv` and `/app/out/wire_return_summary.txt`; `EXCEPTION` rows must leave the `reason` column as an empty CSV field (nothing between commas, not a space character).

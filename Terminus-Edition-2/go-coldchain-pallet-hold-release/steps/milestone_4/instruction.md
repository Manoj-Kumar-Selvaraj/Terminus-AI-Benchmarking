Milestone 4 adds `/app/config/band_policy.csv` with columns `temp_band,enabled,priority`. Only canonical temperature bands whose policy row has `enabled` equal to `Y` are match-eligible. Policy names are canonical after trim and case folding. A release with a concrete `temp_band` such as `FROZEN`, `CHILL`, `AMBIENT`, or an alias must still match the source canonical band exactly and must also pass the enabled-policy gate.

A release with `temp_band` equal to `ANY` is a policy-driven release: it may match any otherwise eligible unused source row whose canonical `temp_band` is enabled. If more than one source row qualifies for an `ANY` release, choose the row with the latest `hold_ts`; if timestamps tie, choose the lowest configured `priority` number; if priority also ties, choose the earliest source input row. Matched `ANY` rows report the selected source row's canonical `temp_band`; never emit `ANY` in the report.

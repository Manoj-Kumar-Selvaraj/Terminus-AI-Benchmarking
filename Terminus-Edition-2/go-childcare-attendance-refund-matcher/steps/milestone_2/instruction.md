Extend the `/app` childcare refund reconciliation CLI to support operator-maintained room aliases. It must still build from `/app/cmd/reconcile`, read `/app/data/sessions.csv` and `/app/data/refunds.csv`, and regenerate `/app/out/refund_report.csv` and `/app/out/refund_summary.json` with the same report and summary schemas.

All prior matching, trimming, header-addressed CSV, malformed-row, full-identifier, positive-amount, status, allowed-room, consumption, ordering, and summary rules still apply. In addition, room normalization is now driven by `/app/config/room_aliases.csv`, which has required columns `alias`, `canonical`, and `enabled`. Read this file at runtime using header-addressed parsing: alias columns may appear in any order and the file may include extra columns. The enabled flag accepts `true`, `y`, `yes`, or `1` after trim and case folding. A row is usable only when `enabled` is true and `canonical` is one of `INFANT`, `TODDLER`, or `PREK` after trim and case folding. If the same alias appears more than once after trim and case folding, the first row for that alias wins and later duplicate rows are ignored. Disabled aliases, blank aliases, blank canonicals, malformed rows, unlisted room values, and aliases pointing to unsupported canonical rooms are ineligible. Malformed alias rows must be skipped without preventing other valid alias rows from being used.

Apply room alias normalization to both session room values and refund room values. The canonical room identities `INFANT`, `TODDLER`, and `PREK` remain valid even if they are not listed in the alias config. Matched report rows must emit the canonical room. Unmatched rows must leave `room` blank. The implementation must honor runtime changes to `/app/config/room_aliases.csv`; do not hardcode only the sample aliases.

Extend the `/app` childcare refund reconciliation CLI to handle dated refund batches. It must still build from `/app/cmd/reconcile`, read `/app/data/sessions.csv` and `/app/data/refunds.csv`, read runtime configs from `/app/config`, and regenerate `/app/out/refund_report.csv` and `/app/out/refund_summary.json` with the same report and summary schemas.

All earlier rules still apply, including header-addressed CSV parsing, runtime room aliases from `/app/config/room_aliases.csv`, canonical room output, positive amounts, full identifier equality, single-use session rows, and deterministic refund input order. Dated input files include `attendance_date` in `sessions.csv` and `refund_date` in `refunds.csv`. Dates use strict `YYYY-MM-DD` format. Missing, blank, or malformed dates are ineligible for matching.

A refund can match only when its `refund_date` is explicitly listed as open in `/app/config/cutoff_calendar.txt` and the refund date is not later than the session `attendance_date`. The cutoff calendar is read at runtime. It contains whitespace-separated `date status` entries; blank lines and lines beginning with `#` are ignored, and only status `open` after trim/case folding makes a date eligible. Closed, unlisted, malformed, or blank calendar dates are not eligible.

If more than one unused session row satisfies all criteria for the same refund, choose the eligible row with the latest `attendance_date`. If attendance dates tie, choose the earliest session input row. Consumption is tracked by session input row position, so duplicate session identifiers in different rows remain separately consumable until their specific rows are used.

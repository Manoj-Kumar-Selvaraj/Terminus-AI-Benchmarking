The order reconciliation CLI in `/app/cmd/reconcile/main.go` is misclassifying same-day marketplace payouts, and the milestone tests fail against the expected matching spec. Fix the Go program so it reads `/app/data/orders.csv` and `/app/data/payouts.csv`, then writes `/app/out/payout_report.csv` and `/app/out/payout_summary.json`. CSV layouts and config file formats are documented in `/app/docs/record_layouts.md`.

A payout matches only when order_id, seller_id, amount, posted order status, and an allowed fulfillment lane all line up; posted order status in `orders.csv` is `SHIPPED`. Canonical lanes are `D2D`, `LOCKER`, and `STORE`. Each order can be matched at most once; if duplicate payouts point at the same order, only the earliest eligible payout consumes it. Trim surrounding spaces on identifiers; lane and status comparisons are case-insensitive.

The report keeps one row per payout in input order with columns `order_id,seller_id,lane,amount_cents,status`. Use `MATCHED` or `UNMATCHED` for status, emit the payout lane when matched, and leave `lane` blank when no order matched. Report fields must not carry incidental surrounding spaces. The summary JSON must contain integer fields `matched_count`, `matched_amount_cents`, `unmatched_count`, and `unmatched_amount_cents`, with payout amounts counted as positive cents.

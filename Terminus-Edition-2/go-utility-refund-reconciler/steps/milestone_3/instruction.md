Extend the bill refund reconciler in `/app/cmd/reconcile/main.go` to handle dated refund batches. It must still read `/app/data/bills.csv` and `/app/data/refunds.csv`, then write `/app/out/refund_report.csv` and `/app/out/refund_summary.json` with the same schema and status values from the earlier milestones. The input files may add a `due_date` column to `bills.csv` and a `refund_date` column to `refunds.csv`. A refund can match only when all prior criteria still pass, the refund date is listed as `open` in `/app/config/cutoff_calendar.txt`, and the refund date is not later than the bill due date. A missing or closed refund date is not eligible. A bill with a missing due date is also not eligible because the refund-date comparison cannot be satisfied. If more than one unused bill matches the same refund, choose the eligible bill with the latest due date; if due dates tie, choose the earliest bill row. Each bill can still be consumed at most once. Legacy aliases from milestone 2 still apply (`CC` means `CARD`, `WIR` means `WIRE`), and matched report rows must emit the canonical channel. Unmatched rows must leave `channel` blank. Summary amounts remain positive integer cents.

Extend the visit reconciliation CLI in `/app/cmd/reconcile/main.go` with policy-driven clinic eligibility. Keep every prior requirement from milestones 1 through 3: full `visit_id`, `owner_id`, amount, `CLOSED` status, alias normalization, row-position consumption, open `credit_date`, `credit_date <= service_date`, latest eligible `service_date`, earliest row tie-break, report schema, status labels, blank unmatched `clinic`, and positive JSON summary totals.

The verifier supplies dated inputs with headers `visit_id,owner_id,amount_cents,status,clinic,service_date` in `/app/data/visits.csv` and `visit_id,owner_id,amount_cents,clinic,credit_date` in `/app/data/credits.csv`. It may rewrite `/app/config/cutoff_calendar.txt` for each scenario.

For this milestone the verifier may rewrite `/app/config/clinic_policy.csv` with columns `clinic,enabled,priority`. Only clinics whose canonical name is present in that file with `enabled` set to `true` may match. Disabled clinics must stay unmatched even when every other gate passes. The `priority` column contains a positive integer; lower numbers rank earlier. Missing or malformed priorities rank after all configured numeric priorities.

Credits may use clinic `ANY`. An `ANY` credit can match any otherwise eligible visit whose canonical clinic is enabled in the policy. For `ANY`, if multiple unused visits qualify, select the visit with the latest `service_date`; when service dates tie, select the enabled clinic with the lower policy priority; when priority also ties, select the earliest visit input row. Non-`ANY` credits still require exact clinic equality after alias normalization (`MN` means `MAIN`, `VAN` means `MOBILE`, `URG` means `ER`) and the clinic must be enabled. Matched rows must emit the selected visit's canonical clinic, never the literal `ANY`. `ANY` credits remain subject to every milestone 3 cutoff-calendar gate: the credit's `credit_date` must be listed as open in `/app/config/cutoff_calendar.txt`, must not be later than the selected visit's `service_date`, and all other dated eligibility rules still apply before policy selection runs.

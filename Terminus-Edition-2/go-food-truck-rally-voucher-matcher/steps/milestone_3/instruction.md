All milestone 1 and 2 rules still apply when date columns are absent; ignore `/app/config/cutoff_calendar.txt` in that legacy mode even if the file is empty.

When both files include `order_date` on orders and `voucher_date` on vouchers, a voucher matches only when prior gates pass, `voucher_date` is listed as `open` in `/app/config/cutoff_calendar.txt` (rows like `YYYY-MM-DD open`, matched case-insensitively; closed or unlisted dates are ineligible), and `voucher_date` is not after `order_date`. Missing `order_date` or `voucher_date` makes that side ineligible. If multiple unused order rows qualify, choose the latest `order_date`; tied dates use the earliest order input row. Track consumption by physical order row position in the input file, not by shared `order_id` or `vendor_id` identity; duplicate order rows may each be consumed by separate vouchers.

The charge reconciliation CLI in `/app/cmd/reconcile/main.go` is misclassifying a few same-day credits. FTL credits are being left unmatched, one charge id collision is matching the wrong row, and the matched total is coming out with the wrong sign.

Fix the Go program so it reads `/app/data/charges.csv` and `/app/data/credits.csv`, then writes `/app/out/credit_report.csv` and `/app/out/credit_summary.json`. A credit matches only when the full `charge_id` equals the bill's `charge_id` (no prefix or partial-id matching), together with `shipper_id`, `amount`, posted charge status `BILLED`, and an allowed mode. Modes `LTL`, `FTL`, and `RAIL` are allowed. Each charge can be matched at most once; if duplicate credits point at the same charge, only the earliest eligible credit consumes it. Input fields can have incidental surrounding spaces, and mode and status comparisons should be case-insensitive.

The report must keep one row per credit in input order with columns `charge_id,shipper_id,mode,amount_cents,status`; use `MATCHED` or `UNMATCHED` for status, use the credit's mode when matched, and leave `mode` blank when no charge matched. Report fields should not carry incidental surrounding spaces from the input. The summary JSON must contain integer fields `matched_count`, `matched_amount_cents`, `unmatched_count`, and `unmatched_amount_cents`, with credit amounts counted as positive cents.

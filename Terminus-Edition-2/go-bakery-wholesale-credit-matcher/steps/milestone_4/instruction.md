Extend the bakery wholesale credit reconciler in `/app/cmd/reconcile/main.go` with route policy controls. The program must still read `/app/data/orders.csv` and `/app/data/credits.csv`, use `/app/config/cutoff_calendar.txt` for dated batches, and write `/app/out/credit_report.csv` and `/app/out/credit_summary.json` with the same exact schemas and `MATCHED` / `UNMATCHED` status values from prior milestones.

All milestone 1 through 3 rules still apply: full `order_id`, `cafe_id`, amount, source status `FULFILLED`, allowed route, legacy aliases (`LOC`, `REG`, `EXP`), single-use order row consumption, open `credit_date`, `credit_date <= bake_date`, latest `bake_date` selection, deterministic credit order, blank route on unmatched rows, and positive summary totals. Matched report rows must emit the canonical credit route for concrete route credits after normalization.

Milestone 4 adds `/app/config/route_policy.csv` with exactly the columns `route,enabled,priority`. Route policy applies to both older undated input schemas and dated input schemas; the cutoff calendar and `credit_date <= bake_date` gates apply only when the dated columns are present. Policy route names and enabled values are both trimmed and compared case-insensitively, so ` regional ` canonicalizes to `REGIONAL` and ` y ` enables that row. Only canonical routes whose policy row has enabled value `Y` after trimming and case folding are match-eligible; every other enabled value is disabled. Policy route names are canonicalized after trimming and case folding before validation. For a concrete credit route such as `LOCAL`, `REGIONAL`, `EXPORT`, or a documented alias, the source order route must still canonicalize to the same route and that route must be enabled by policy.

A credit with route `ANY` is a policy-driven wildcard. It may match any otherwise eligible unused order row whose canonical route is enabled. Disabled routes and unconfigured routes with no valid policy row are ineligible for both concrete credits and `ANY` credits. Matched `ANY` rows must report the selected order row's canonical route; never emit `ANY`. If more than one source row qualifies for an `ANY` credit, choose the latest `bake_date`; if those tie, choose the lowest configured priority number; if priority also ties, choose the earliest order input row.

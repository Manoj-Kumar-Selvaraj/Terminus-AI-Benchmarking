Extend the Go booking refund reconciliation CLI to support dated refund batches while preserving exact matching, runtime alias normalization, amount validation, booking-row consumption, report formatting, and summary totals from earlier milestones.

Legacy files without date columns must continue to reconcile without consulting `/app/config/cutoff_calendar.txt`. Duplicate booking rows remain separate consumable rows by physical input position. Dated files may add booking header `slot_date` and refund header `refund_date`; when either file includes its date column, date eligibility is active and rows missing the counterpart date are ineligible. Dates must be strict valid `YYYY-MM-DD`. For dated matching, the refund date must be listed as open in `/app/config/cutoff_calendar.txt`, where well-formed lines are `YYYY-MM-DD STATE` and only case-insensitive `OPEN` is eligible. Ignore blank, comment, malformed, unlisted, closed, and invalid calendar rows. The refund date must be on or before the booking slot date.

When multiple unused booking rows satisfy every gate for the same refund, select the eligible booking with the latest `slot_date`. When that date ties, select the earliest unused booking input row. Track consumption by physical booking row, not by `booking_id`. Keep output schemas, status vocabulary, canonical matched room-tier values, blank unmatched room tier, preserved amount text, and numeric summary fields unchanged.

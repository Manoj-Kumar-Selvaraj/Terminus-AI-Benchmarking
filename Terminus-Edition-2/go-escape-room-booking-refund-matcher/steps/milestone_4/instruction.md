Extend the Go booking refund reconciliation CLI with refund-reason eligibility, runtime room-tier policy, and `ANY` room-tier refunds while preserving all earlier milestone behavior.

When `refunds.csv` includes `refund_reason`, read `/app/config/refund_reasons.csv` at runtime by header name using `reason` and `eligible`. Trim and case-fold reason values. Only reasons whose last well-formed duplicate row has `eligible = Y` case-insensitively are eligible. Treat blank or missing refund reasons and reasons not marked eligible as ineligible when the column is present.

Read `/app/config/room_tiers.csv` by header name using `tier`, `enabled`, and optional `priority`. Trim and case-fold tier names, support only canonical `EASY`, `HARD`, and `VIP`, and allow tiers whose last well-formed row has boolean `enabled = true`. Use lower numeric `priority` as higher priority; treat missing or malformed priority as lower rank than numeric priorities. From this step onward, a refund room tier of `ANY` matches any enabled canonical booking tier that satisfies every other gate. Never emit `ANY` in matched report rows; emit the selected canonical booking tier. For `ANY` candidates, rank by latest `slot_date` when dated matching is active, then lower configured tier priority, then earliest unused booking input row. `ANY` still passes identity, team, amount, booking status, reason, alias, date, calendar, and consumption gates.

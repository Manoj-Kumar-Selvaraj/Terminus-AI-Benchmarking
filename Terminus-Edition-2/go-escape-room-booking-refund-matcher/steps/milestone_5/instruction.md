Add team-level refund policy enforcement to the Go booking refund reconciliation CLI while preserving exact identity gates, runtime aliases, date and calendar logic, refund reasons, room-tier policy, `ANY` ranking, output schemas, amount preservation, and booking-row consumption from earlier milestones.

Read `/app/config/team_limits.csv` at runtime by header name using `team_id`, `room_tier`, `max_refund_cents`, `enabled`, and `allow_any`; extra columns may appear. A well-formed policy row has nonblank trimmed `team_id`, a room tier that normalizes through the runtime alias map to canonical `EASY`, `HARD`, or `VIP`, a positive base-10 integer `max_refund_cents`, and boolean `enabled` and `allow_any` values. Ignore blank, comment-like, incomplete, malformed, nonboolean, nonnumeric, zero, negative, unsupported-tier, and `ANY` policy rows. When the file contains no well-formed policies, no team policy gate is active. For duplicate `team_id` plus canonical `room_tier` keys, the last well-formed row is authoritative.

When policies are active, a candidate matches only if its refund `team_id` and selected canonical booking room tier share an exact policy key, the authoritative policy is enabled, the refund amount is within `max_refund_cents`, and `allow_any = true` when the refund requested `ANY`. Policy-blocked candidates must not consume booking rows.

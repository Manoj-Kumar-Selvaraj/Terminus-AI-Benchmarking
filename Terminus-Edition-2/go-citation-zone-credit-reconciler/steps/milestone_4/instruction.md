Extend the citation reconciliation CLI in `/app/cmd/reconcile/main.go` to handle credit method policy gating. It must still read `/app/data/citations.csv` and `/app/data/credits.csv`, then write `/app/out/credit_report.csv` and `/app/out/credit_summary.json` with the same schema and status values from the earlier milestones. All milestone 1, 2, and 3 behavior remains required: full identifiers, plate_id, amount, citation status `PAID` only, rejection of `POSTED`, canonical zones, single-use citation row consumption, credit input order, positive summary totals, aliases (`ST` means `STREET`, `GRG` means `GARAGE`, `LT` means `LOT`), open credit dates from `/app/config/cutoff_calendar.txt`, `credit_date` not later than `due_date`, latest eligible due_date selection, and earliest citation row tie-breaking.

The input files may add a `credit_method` column to `credits.csv`. When that column is present, a credit can match only if its method is enabled in `/app/config/methods.csv`. The methods config uses CSV columns `method,enabled`. Compare method names after trimming and case folding. A method is enabled only when the config row's `enabled` value is `true`, also after trimming and case folding. Disabled methods, methods missing from the config, blank credit methods, blank config methods, malformed config rows, or non-true enabled values such as `yes`, `1`, `enabled`, or blank make the credit ineligible. If `credit_method` is absent from the credit input header, preserve milestone 3 behavior without method gating. Method gating must compose with all prior gates: an enabled method alone must not allow a credit to match when identity, amount, citation status, zone, date, calendar, or consumption rules fail. The report schema remains `citation_id,plate_id,zone,amount_cents,status`; do not add a method column. Matched rows still emit the canonical zone, and unmatched rows still leave `zone` blank.

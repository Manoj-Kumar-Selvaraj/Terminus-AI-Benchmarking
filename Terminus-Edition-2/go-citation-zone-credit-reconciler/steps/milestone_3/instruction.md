Extend the citation reconciliation CLI in `/app/cmd/reconcile/main.go` to handle dated credit batches. It must still read `/app/data/citations.csv` and `/app/data/credits.csv`, then write `/app/out/credit_report.csv` and `/app/out/credit_summary.json` with the same schemas and positive cent totals as earlier milestones.

The input files may add a `due_date` column to `citations.csv` and a `credit_date` column to `credits.csv`. A credit can match only when all prior criteria still pass, including citation status `PAID`, the `credit_date` is listed as `open` in `/app/config/cutoff_calendar.txt`, and the `credit_date` is not later than the citation `due_date`. A missing or closed `credit_date` is not eligible, and a citation with a missing `due_date` is also not eligible. If both input files omit the date columns, the program must still read them without crashing, but those rows are not eligible for dated matching and should be reported as unmatched. If more than one unused citation matches the same credit, choose the eligible citation with the latest `due_date`. Consumption is tracked by citation input row position, not by `citation_id`, so duplicate ids in separate rows remain independently consumable until their specific rows are used. Legacy aliases from milestone 2 still apply (`ST` means `STREET`, `GRG` means `GARAGE`, `LT` means `LOT`); matched report rows must emit the canonical zone and unmatched rows must leave `zone` blank.

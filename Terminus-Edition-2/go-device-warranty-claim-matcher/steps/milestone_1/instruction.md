The device warranty reconciliation CLI in `/app/cmd/reconcile/main.go` is misclassifying a few same-day claims. BATTERY claims are being left unmatched, one device id collision is matching the wrong row, and the matched total is coming out with the wrong sign. Fix the Go program so it reads `/app/data/devices.csv` and `/app/data/warranty_claims.csv`, then writes `/app/out/warranty_report.csv` and `/app/out/warranty_summary.json`. A claim matches only when device id, owner id, amount, posted device status, and an allowed reason all line up. Reasons `SCREEN`, `BATTERY`, and `WATER` are allowed. Each device can be matched at most once; if duplicate claims point at the same device, only the earliest eligible claim consumes it. Input fields can have incidental surrounding spaces, and reason and status comparisons should be case-insensitive. The report must keep one row per claim in input order with columns `device_id,owner_id,reason,amount_cents,status`; use `MATCHED` or `UNMATCHED` for status, use the claim's reason when matched, and leave `reason` blank when no device matched. Report fields should not carry incidental surrounding spaces from the input. The summary JSON must contain integer fields `matched_count`, `matched_amount_cents`, `unmatched_count`, and `unmatched_amount_cents`, with claim amounts counted as positive cents.

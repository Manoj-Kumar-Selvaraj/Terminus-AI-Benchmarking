For this milestone, the verifier may add a `screening_date` column to `screenings.csv` and a `credit_date` column to `credits.csv`. A credit can match only when all prior criteria still pass, the credit_date is listed as `open` in `/app/config/cutoff_calendar.txt`, and the credit_date is not later than the screening screening_date. A missing or closed credit_date is not eligible. A screening with a missing screening_date is also not eligible because the date comparison cannot be satisfied. If more than one unused screening matches the same credit, choose the eligible screening with the latest screening_date; if screening_dates tie, choose the earliest screening row. Consumption is tracked by screening input row position, not by screening_id, so duplicate ids in separate rows remain independently consumable until their specific rows are used.

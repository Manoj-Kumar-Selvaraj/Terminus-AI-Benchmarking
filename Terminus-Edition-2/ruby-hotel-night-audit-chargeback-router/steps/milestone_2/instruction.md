Milestone 2 keeps every milestone 1 rule and adds legacy kind alias normalization before matching. Normalize correction and source `kind` values after trimming and case folding: `CC` maps to `CARD`, `CSH` maps to `CASH`, and `PTS` maps to `POINTS`. After normalization, only canonical values `CARD`, `CASH`, or `POINTS` are match-eligible; unknown normalized values stay unmatched even when source and correction share the same unknown token. Matched report rows must emit the canonical kind from the matched source row, never the raw alias.

All milestone 1 identity, window, timestamp, consumption, reason, and output rules still apply. Preserve correction input order, use only `MATCHED` or `UNMATCHED`, leave unmatched `kind` blank, and keep the milestone 1 report and summary schemas unchanged.

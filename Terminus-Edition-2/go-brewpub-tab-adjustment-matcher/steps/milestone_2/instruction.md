Update the tab reconciliation CLI in `/app` for a legacy adjustment export that uses pour tier aliases. The program must still read `/app/data/tabs.csv` and `/app/data/adjustments.csv`, then write `/app/out/tab_adjustment_report.csv` and `/app/out/tab_adjustment_summary.json`. A tab adjustment matches only when `tab_id`, `patron_id`, `amount_cents`, eligible tab status `COMPLETED` (compared case-insensitively), and allowed pour tier line up. Tab ids must be compared as full identifiers. Legacy adjustment pour tier aliases `PT`, `PC`, and `KG` normalize after trimming and case folding to canonical `PINT`, `PITCH`, and `KEG`. See `/app/docs/support_matrix.md` for the alias table. Keep the deliverable as a Go CLI: the verifier compiles the Go source with the Go toolchain available at `/usr/local/go/bin/go` and then runs the produced binary.

Report rows appear in adjustment input order (one row per adjustment), as established in milestone 1. Each tab row can be consumed by at most one adjustment, with the earliest eligible adjustment winning. Header-only `tabs.csv` or `adjustments.csv` behavior from milestone 1 still applies. The verifier expects `tab_adjustment_report.csv` to begin with `tab_id,patron_id,pour_tier,amount_cents,status`; matched rows emit canonical pour tiers, unmatched rows leave `pour_tier` blank, and report identifier fields must not carry incidental spaces from adjustment input. The summary JSON must contain only integer-typed fields `matched_count`, `matched_amount_cents`, `unmatched_count`, and `unmatched_amount_cents`, with positive cent totals.

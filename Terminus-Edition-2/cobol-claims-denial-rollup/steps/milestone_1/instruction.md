The denial adjustment rollup in `/app/src/claim_rollup.cbl` is producing bad finance handoff files. COB-related denials are being skipped, some claim ids collide on the wrong denial record, and the matched denial total is negative.

Fix the COBOL program so it reads `/app/data/claims.dat` and `/app/data/adjustments.dat`, then writes `/app/out/denial_report.csv` and `/app/out/denial_summary.txt`. An adjustment matches only when claim id, member id, amount, denied claim status, and an allowed denial reason all line up. Reasons `MED`, `NEC`, `COB`, and `AUT` are allowed. Each denied claim can be matched at most once; duplicate adjustments for the same claim leave only the earliest eligible adjustment matched.

The report must keep one row per adjustment in input order with columns `claim_id,member_id,reason,amount_cents,status`; use `MATCHED` or `UNMATCHED` for status and leave `reason` blank when no denial matched. In the report, `amount_cents` should stay as the 10-character zero-padded cents value from the adjustment record, like `0000000300`. The summary should use `key=value` lines for `matched_count`, `matched_amount_cents`, `unmatched_count`, and `unmatched_amount_cents`, with adjustment amounts counted as positive cents.

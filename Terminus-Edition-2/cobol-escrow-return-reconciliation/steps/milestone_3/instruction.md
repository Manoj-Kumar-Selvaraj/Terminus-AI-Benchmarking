Finish the wire-return batch in `/app/src/wire_returns.cbl` for settlement-cycle controls. Wire and return records are fixed-width 64-byte lines. See `/app/docs/record_layouts.md` for byte offsets; milestone 3 extends both layouts with 8-character dates.

Wire layout: byte 1 is `W`; bytes 2–13 `wire_id`; 14–16 `reason`; 17–26 zero-padded `amount_cents`; 27–34 `account_id`; 35 `status` (`S` settled); bytes 36–43 optional `settlement_date` (`YYYYMMDD`). Return layout: byte 1 is `R`; bytes 2–13 `wire_id`; 14–23 `amount_cents`; 24–31 `account_id`; bytes 32–39 optional `return_date` (`YYYYMMDD`).

Use `/app/config/cycle_calendar.txt`, where only rows marked `OPEN` are settlement-cycle days. Calendar status tokens are case-insensitive (`open`, `Open`, and `OPEN` all mean open), and calendar rows may appear in any order.

A return can clear only when the wire settlement date and return date are nonblank, both appear in the calendar with status OPEN (not merely listed), the return date is not before the wire settlement date, and the return arrives within two open cycle days after the wire settlement date. Count open days after the wire settlement date through the return date, including the return date if it is open and excluding closed or absent calendar dates. Exactly two open cycle days is still eligible; three or more is not. Dates missing from the calendar are not open, and a blank settlement date or blank return date must be treated as ineligible rather than as a same-day match. If more than one unused wire satisfies all matching and cycle-window rules for a return, clear against the eligible wire with the latest settlement date; when dates tie, use the earliest wire input row. Keep all prior behavior and output schemas for `/app/out/wire_return_report.csv` and `/app/out/wire_return_summary.txt`; `EXCEPTION` rows must leave the `reason` field blank.

Milestone 1 through milestone 3 behavior must remain correct while adding runtime payment-method policy, `ANY` payments, customer daily limits, and an audit artifact. Keep the deliverable as a Go CLI compiled with `/usr/local/go/bin/go`.

Read `/app/config/methods.csv` at runtime. Its header is `method,enabled,priority`. Method names are canonical `ACH`, `CARD`, or `WIRE`; surrounding whitespace is ignored. `enabled=true` is case-insensitive and whitespace-tolerant. Disabled, missing, malformed, or non-canonical method rows are ineligible. Lower numeric `priority` values rank earlier. If a canonical method has multiple rows, the last valid row for that method is authoritative.

A payment method of `ANY` is a wildcard. It must never be emitted in `/app/out/payment_report.csv`; matched `ANY` payments emit the selected invoice's canonical method. For dated inputs, wildcard candidates are ranked after all normal eligibility filters by latest eligible `due_date`, then lower method priority, then earliest invoice row. Non-wildcard payments still require canonical method equality. Customer limits are read from `/app/config/customer_limits.csv` with header `customer_id,method,max_amount_cents,effective_date,enabled`. Limits apply by cleaned customer, selected canonical method, and payment date. A limit row is applicable only when `enabled=true`, the limit amount is a non-negative integer, and `effective_date` is not after the payment date. When multiple rows apply, use the latest `effective_date`; ties use the later file row. If no limit applies, the budget is unlimited. Limit budget is consumed in payment input order and is independent per customer, selected canonical method, and payment date.

Also write `/app/out/payment_audit.csv` with columns `method,total_payments,matched_count,unmatched_count,matched_amount_cents,unmatched_amount_cents`. Group by the canonical requested method for normal payments, by the selected canonical method for matched `ANY` payments, and by `ANY` for unmatched wildcard payments. Emit audit rows in this order for groups that appear: `ACH`, `CARD`, `WIRE`, `ANY`, followed by any other groups lexicographically. The report and summary schemas from earlier milestones remain unchanged.

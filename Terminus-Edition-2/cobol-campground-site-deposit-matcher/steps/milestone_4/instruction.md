Extend the campground site deposit reconciler in `/app/src/camp_deposit_reconcile.cbl` and supporting source under `/app/src` with runtime reason and category policy. Preserve all milestone 1 through 3 behavior.

Read `/app/config/reasons.csv` at runtime by header name using columns `code` and `eligible`. Trim and uppercase reason codes. A reason is eligible only when the last well-formed row for that code has `eligible = Y` case-insensitively. Blank or malformed rows are ignored. Do not hardcode the earlier eligible reason set after this milestone.

Read `/app/config/categories.csv` at runtime by header name using columns `code`, `enabled`, and optional `priority`. Trim and uppercase category codes. A canonical source category is eligible only when the last well-formed row for that code has `enabled = true` case-insensitively. Invalid enabled values, blank codes, and malformed rows are ignored. Category `priority` is an integer; lower numbers rank earlier, and missing or malformed priority ranks after numeric priorities. Canonical source categories must be configured and enabled before they can match.

Introduce action `site_class` value `ANY` in this milestone. `ANY` is a wildcard over enabled canonical source categories only; it does not bypass record id, account, branch, amount, source status, reason, calendar, date ordering, or source-consumption gates. Matched `ANY` rows emit the selected canonical source `site_class`, never `ANY`. For `ANY` candidates, select by latest eligible source date first, then lower configured category priority, then earliest source input row.

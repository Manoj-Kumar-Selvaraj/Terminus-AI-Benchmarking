Now apply `/app/config/season_calendar.txt` to source dates. The calendar is a runtime text file containing `YYYYMMDD=STATE` rows; blank rows, comment rows beginning with `#`, malformed rows, unlisted dates, and missing dates are not eligible. A source date is eligible only when the source date is numeric 8-digit text, appears in the calendar, and its state equals `OPEN` case-insensitively, including mixed-case values such as `oPeN` and `opEN`. Closed, malformed, missing, unlisted, or nonnumeric source dates are ineligible. The action date must still be numeric 8-digit text and `action_date >= source_date`; the action date itself does not need to be open in the calendar.

When more than one unused source row satisfies all identity, amount, branch, status, reason, alias, date, and calendar gates for the same action, choose the eligible source row with the latest `source_date`. If the latest dates tie, choose the earliest source input row. Track consumption by source row position, not by `record_id` alone.

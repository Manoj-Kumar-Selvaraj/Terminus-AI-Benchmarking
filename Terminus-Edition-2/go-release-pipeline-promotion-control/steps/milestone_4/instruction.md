Operators invoke `go run ./cmd/pipelinesim rollback --history PATH --env ENV --out DIR [--target-build BUILD]`, but `/app/evidence/rollback_terminal_capture.txt` shows rollback completing by rebuilding from current branch state instead of redeploying an artifact from release history. Review `/app/docs/rollback_contract.md` and `/app/evidence/release_history_excerpt.json`.

Restore rollback from immutable release history. The command interface, environment names, `rollback_manifest.json` schema, and promoted artifact hash field must remain stable. Both selection modes may use only records whose `promotion_status` is exactly `promoted`. For default rollback, filter to promoted records for the requested environment, exclude the most recent promoted record, then pick the latest of the remaining promoted records. Failed, pending, or other non-promoted records never participate in this selection. An explicit `--target-build` must select a promoted record from that same environment; reject wrong-environment, entirely absent, or non-promoted targets without writing `rollback_manifest.json`. Malformed or missing history must also fail without writing `rollback_manifest.json`.

Rollback eligibility is additionally fenced by the environment compatibility floor in `/app/config/pipeline_policy.json` at `rollback.minimum_release_contract_by_env`. Only **promoted** records participate in compatibility filtering. A promoted record is eligible only when its `release_contract_version` is present, well-formed in `YYYY.MM` form, and greater than or equal to the configured floor for the requested environment. Records with `failed`, `pending`, or any other non-promoted status are ignored for rollback selection and must not cause the history file to be rejected merely because they omit or malform `release_contract_version`. Apply the compatibility filter to promoted records before default rollback excludes the most recent eligible promoted record. Explicit `--target-build` requests for promoted records below the environment floor must fail closed without writing a rollback manifest.

The rollback manifest must set `rollback_source` to the literal `release_history` and include `command_interface` with `pipelinesim rollback --history PATH --env ENV --out DIR [--target-build BUILD]`. Copy `artifact_hash` from the selected history record's `artifact_hash` field and copy `promoted_artifact_hash` independently from that record's `promoted_artifact_hash` field; these values may differ and must not be substituted for one another. Preserve the artifact identity, workspace isolation, and quality-gate protections already established.
